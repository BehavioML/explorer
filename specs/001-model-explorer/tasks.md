# Model Explorer Tasks

## Status

Implementation tasks are not ready.

This file exists as a Spec Kit-inspired placeholder for future task planning. It intentionally does not define implementation tasks, task ordering, file-level changes, framework setup, package installation, or code scaffolding.

## Readiness gates

Implementation tasks should not be created until:

- the source spec has been reviewed;
- the BehavioML draft has been derived;
- model gaps have been resolved or explicitly documented;
- validation and review reports exist where useful;
- a technical plan exists;
- relevant contracts or data-model decisions exist where needed.

## Future task sources

Future tasks should be derived from:

- reviewed source requirements;
- accepted or reviewed BehavioML workflows, capabilities, interfaces, events, entities, state machines, and decisions;
- technical planning decisions in `plan.md` and `research.md`;
- contracts under `contracts/` when they exist;
- data-model decisions in `data-model.md` when they exist.

## Non-goals for this placeholder

- Do not create application implementation tasks yet.
- Do not choose a frontend framework through tasks.
- Do not add package setup tasks.
- Do not add code-generation tasks before the behavior model is reviewed.
- Do not use tasks to invent behavior missing from source specs or BehavioML.
